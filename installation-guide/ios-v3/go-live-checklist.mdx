# Go-Live Checklist

Use this checklist to ensure your Gameball iOS SDK v3.0.0 integration is complete and ready for production.

<Info>
During development, use your account **Test Key** to authenticate requests. Once integration is completed and validated, swap to your **Production API Key**.
</Info>

## Step 1: Install SDK and Verify Dependencies

- [ ] Gameball SDK v3.0.0 is installed via CocoaPods or SPM
- [ ] All dependencies are resolved without conflicts
- [ ] SDK imports correctly in your view controllers
- [ ] No compilation errors or warnings
- [ ] `Gameball.shared.initialize(with:)` succeeds

<Tip>
For information, please read more at [Getting Started](/installation-guide/ios-v3/getting-started)
</Tip>

## Step 2: Initialize and Register Customers

- [ ] SDK initializes successfully with your API key in AppDelegate or App struct
- [ ] Test customer registration with `initializeCustomer` returns success
- [ ] Customer data appears correctly in Gameball dashboard
- [ ] Error handling works for invalid registrations
- [ ] Async/await methods work correctly (if using iOS 15+)

<Tip>
For information, please read more at [Initialize Customer Profile](/installation-guide/ios-v3/initialize-profile)
</Tip>

## Step 3: Track Events

- [ ] Test events are sent successfully using `sendEvent`
- [ ] Events appear in Gameball dashboard with correct metadata
- [ ] Event-based rewards trigger properly
- [ ] Error handling works for failed API calls
- [ ] Both callback and async/await methods work correctly

<Tip>
For information, please read more at [Track Customer Events](/installation-guide/ios-v3/track-events)
</Tip>

## Step 4: Track Orders and Cashback

- [ ] Test orders are tracked using `sendOrder`
- [ ] Orders appear in Gameball dashboard with complete data
- [ ] Cashback calculations are accurate
- [ ] Points are awarded correctly to customer accounts
- [ ] Refunds properly adjust point balances

<Tip>
For more information, please read more at [Track Orders & Cashback](/installation-guide/ios-v3/track-orders)
</Tip>

## Step 5: Show Profile Widget

- [ ] Profile widget displays correctly
- [ ] Customer data loads properly in widget
- [ ] Navigation works within the widget
- [ ] Close button functions correctly (if enabled)
- [ ] Widget displays on both UIKit and SwiftUI implementations
- [ ] Deep linking to specific sections works

<Tip>
For more information, please read more at [Show Profile Widget](/installation-guide/ios-v3/show-profile)
</Tip>

## Step 6: Redemption Integration

- [ ] Customer balance API works correctly
- [ ] Customers can view their points balance
- [ ] Redemption flow is integrated with your backend
- [ ] Points are deducted correctly upon redemption
- [ ] UI updates properly after redemption

<Tip>
For more information, please read more at [Integrate Redemption](/installation-guide/ios-v3/integrate-redemption)
</Tip>

## Step 7: Push Notifications

- [ ] APNs certificate is configured correctly
- [ ] Device tokens are generated and registered successfully
- [ ] Push notification credentials configured in Gameball dashboard
- [ ] Test notifications are received on devices
- [ ] Notification content displays correctly
- [ ] Token refresh handling works properly
- [ ] Notification delegate methods implemented correctly

<Tip>
For more information, please read more at [Push Notifications](/installation-guide/ios-v3/push-notifications)
</Tip>

## Step 8: Referrals Tracking

- [ ] Firebase Dynamic Links SDK is integrated
- [ ] Dynamic Links are configured in Firebase console
- [ ] Firebase configuration added to Gameball dashboard
- [ ] App correctly captures referral codes from deep links
- [ ] Referral codes are included during customer registration
- [ ] Both referrer and referee are rewarded correctly
- [ ] Info.plist configured for URL schemes

<Tip>
For information, please check [Track Referrals](/installation-guide/ios-v3/track-referrals)
</Tip>

## Step 9: Production Readiness

- [ ] Swap Test API Key to Production API Key
- [ ] App works correctly on all supported iOS versions (13.0+)
- [ ] Test on multiple iOS devices (iPhone, iPad)
- [ ] Test on both iOS 13-16 and iOS 17+
- [ ] Memory usage is within acceptable limits
- [ ] No memory leaks in long-running sessions
- [ ] Crash reporting is configured and working
- [ ] App Store submission requirements met

## Step 10: Testing & Validation

- [ ] Test on real devices, not just simulator
- [ ] Verify functionality in both Debug and Release builds
- [ ] Test with poor network conditions
- [ ] Verify all error scenarios are handled
- [ ] Test deep linking and URL schemes
- [ ] Validate data synchronization with Gameball dashboard
- [ ] Performance testing completed

## Step 11: Monitoring and Support

- [ ] Monitor API response times after launch
- [ ] Check crash reports for any SDK-related issues
- [ ] Verify customer transactions in Gameball dashboard
- [ ] Support team is ready to handle inquiries
- [ ] Documentation is accessible to team
- [ ] Analytics and monitoring tools configured

<Check>
Once you've completed all items in this checklist, your iOS Gameball SDK v3.0.0 integration is ready for production!
</Check>
