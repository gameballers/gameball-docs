# Go-Live Checklist

Use this checklist to ensure your Gameball React Native SDK v3.1.0 integration is complete and ready for production.

<Info>
During development, use your account **Test Key** to authenticate requests. Once integration is completed and validated, swap to your **Production API Key**.
</Info>

## Step 1: Install SDK and Dependencies

- [ ] Gameball React Native SDK v3.1.0 is installed via npm/yarn
- [ ] `react-native-webview` dependency is installed
- [ ] All dependencies are resolved without conflicts
- [ ] SDK imports correctly in your TypeScript/JavaScript files
- [ ] No compilation errors or warnings
- [ ] iOS pods are installed successfully (`cd ios && pod install`)
- [ ] Android build completes without errors

<Tip>
For information, please read more at [Getting Started](/installation-guide/react-native-v3/getting-started)
</Tip>

## Step 2: Initialize SDK

- [ ] SDK initializes successfully with your API key
- [ ] `GameballApp.getInstance().init(config)` runs without errors
- [ ] Language and platform settings are configured correctly
- [ ] Config includes all required parameters (apiKey, lang)

<Tip>
For information, please read more at [Initialize SDK](/installation-guide/react-native-v3/initialize-sdk)
</Tip>

## Step 3: Register and Display Customer Profile

- [ ] Test customer registration with `initializeCustomer` returns success
- [ ] Customer data appears correctly in Gameball dashboard
- [ ] Profile widget displays correctly on both iOS and Android
- [ ] Widget navigation works properly
- [ ] Close button functions correctly
- [ ] Error handling works for invalid registrations

<Tip>
For information, please read more at [Initialize Customer Profile](/installation-guide/react-native-v3/initialize-profile)
</Tip>

## Step 4: Track Events

- [ ] Test events are sent successfully using `sendEvent`
- [ ] Events appear in Gameball dashboard with correct metadata
- [ ] Event-based rewards trigger properly
- [ ] Error handling works for failed API calls
- [ ] Both async/await and callback patterns work

<Tip>
For information, please read more at [Track Customer Events](/installation-guide/react-native-v3/track-events)
</Tip>

## Step 5: Track Orders and Cashback

- [ ] Server-side Order API integration is working
- [ ] Orders appear in Gameball dashboard with complete data
- [ ] Cashback calculations are accurate
- [ ] Points are awarded correctly to customer accounts
- [ ] Refunds properly adjust point balances

<Tip>
For more information, please read more at [Track Orders & Cashback](/installation-guide/react-native-v3/track-orders)
</Tip>

## Step 6: Redemption Integration

- [ ] Customer balance can be retrieved via server API
- [ ] Customers can view their points balance
- [ ] Redemption flow is integrated with your backend
- [ ] Points are deducted correctly upon redemption
- [ ] UI updates properly after redemption

<Tip>
For more information, please read more at [Integrate Redemption](/installation-guide/react-native-v3/integrate-redemption)
</Tip>

## Step 7: Referrals Tracking

- [ ] `@react-native-firebase/dynamic-links` is installed
- [ ] Dynamic Links are configured in Firebase console
- [ ] Firebase configuration added to Gameball dashboard
- [ ] App correctly captures referral codes from deep links
- [ ] Referral codes are included during customer registration
- [ ] Both referrer and referee are rewarded correctly

<Tip>
For information, please check [Track Referrals](/installation-guide/react-native-v3/track-referrals)
</Tip>

## Step 8: Production Readiness

- [ ] Swap Test API Key to Production API Key in `GameballConfig`
- [ ] Test on multiple iOS devices and versions (iOS 12.0+)
- [ ] Test on multiple Android devices and versions (API 21+)
- [ ] Test on different screen sizes and orientations
- [ ] ProGuard rules added for Android release builds
- [ ] Memory usage is within acceptable limits
- [ ] No memory leaks in long-running sessions
- [ ] Crash reporting is configured and working

## Step 9: Platform-Specific Checks

### iOS
- [ ] iOS deployment target is 12.0 or higher
- [ ] Pods are installed and up-to-date
- [ ] App Store privacy requirements are met
- [ ] Info.plist permissions are properly configured

### Android
- [ ] minSdkVersion is 21 or higher
- [ ] targetSdkVersion is 34
- [ ] ProGuard rules don't conflict with Gameball SDK
- [ ] AndroidManifest.xml has INTERNET permission

## Step 10: Monitoring and Support

- [ ] Monitor API response times after launch
- [ ] Check crash reports for any SDK-related issues
- [ ] Verify customer transactions in Gameball dashboard
- [ ] Support team is ready to handle inquiries
- [ ] Documentation is accessible to team

<Check>
Once you've completed all items in this checklist, your React Native Gameball SDK v3.1.0 integration is ready for production!
</Check>
