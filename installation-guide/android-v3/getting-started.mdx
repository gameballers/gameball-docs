# Getting Started

## Requirements

- **Minimum SDK Version**: 21 (Android 5.0)
- **Target SDK Version**: 34
- **Kotlin**: 2.0.0+
- **AndroidX**: Required
- **Internet permission** in `AndroidManifest.xml`

```xml
<uses-permission android:name="android.permission.INTERNET" />
```

## Install the SDK

### Gradle (Recommended)

Add required repositories to your project-level `settings.gradle` or `build.gradle`:

```groovy
dependencyResolutionManagement {
  repositories {
    google()
    mavenCentral()
    maven { url "https://jitpack.io" }
    maven { url 'https://developer.huawei.com/repo/' }
  }
}
```

Add the dependency to your app-level `build.gradle`:

```groovy
dependencies {
  implementation 'com.gameball:gameball-sdk:3.1.0'
}
```

<Info>
If your project uses older Gradle settings, replace `dependencyResolutionManagement` usage accordingly.
</Info>

### Maven

Add repositories to `pom.xml` then the dependency:

```xml
<dependency>
  <groupId>com.gameball</groupId>
  <artifactId>gameball-sdk</artifactId>
  <version>3.1.0</version>
</dependency>
```

## Initialize the SDK

Create an application-level initialization so the SDK is available globally.

<CodeGroup>
```kotlin Kotlin
class MyApplication : Application() {
    override fun onCreate() {
        super.onCreate()

        val config = GameballConfig.builder()
            .apiKey("your-api-key")      // Required
            .lang("en")                  // Required
            .platform("android")         // Optional (defaults to android)
            .shop("your-shop-id")        // Optional
            .gbToken("your-gb-token")    // Optional - for v4.1 endpoints
            .apiPrefix("custom-prefix")  // Optional - custom API prefix
            .build()

        GameballApp.getInstance(this).init(config)
    }
}
```

```java Java
public class MyApplication extends Application {
    @Override
    public void onCreate() {
        super.onCreate();

        GameballConfig config = GameballConfig.builder()
            .apiKey("your-api-key")      // Required
            .lang("en")                  // Required
            .platform("android")         // Optional (defaults to android)
            .shop("your-shop-id")        // Optional
            .gbToken("your-gb-token")    // Optional - for v4.1 endpoints
            .apiPrefix("custom-prefix")  // Optional - custom API prefix
            .build();

        GameballApp.getInstance(this).init(config);
    }
}
```
</CodeGroup>

## Configuration Parameters

<ParamField body="apiKey" type="string" required>
The client API key available in your Gameball dashboard (required).
</ParamField>

<ParamField body="lang" type="string" required>
The language code for SDK localization (e.g., "en", "ar", "fr"). Required.
</ParamField>

<ParamField body="platform" type="string">
Platform identifier, defaults to "android".
</ParamField>

<ParamField body="shop" type="string">
Your Gameball shop identifier for multi-shop configurations.
</ParamField>

<ParamField body="gbToken" type="string">
GB Token for secure authentication (enables automatic v4.1 endpoint routing).
</ParamField>

<ParamField body="apiPrefix" type="string">
Custom API endpoint prefix.
</ParamField>

<Tip>
Make sure to register your Application class in AndroidManifest.xml:
```xml
<application
    android:name=".MyApplication"
    ...>
```
</Tip>

## ProGuard/R8 Configuration

If you're using code obfuscation, add these rules to your `proguard-rules.pro`:

```proguard
# Gameball SDK
-keep class com.gameball.gameball.** { *; }
-keep class com.gameball.gameball.model.** { *; }

# Gson
-keepattributes Signature
-keepattributes *Annotation*
-keep class com.google.gson.** { *; }
```

<Info>
The SDK automatically enables debug logging in debug builds. Network requests and responses are logged when `BuildConfig.DEBUG` is true.
</Info>
