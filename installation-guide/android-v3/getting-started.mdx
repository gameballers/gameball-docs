# Getting Started

## Requirements

- **Minimum SDK Version**: 21 (Android 5.0)
- **Target SDK Version**: 34
- **Kotlin**: 2.0.0+
- **AndroidX**: Required
- **Internet permission** in `AndroidManifest.xml`

```xml
<uses-permission android:name="android.permission.INTERNET" />
```

## Install the SDK

### Gradle (Recommended)

Add required repositories to your project-level `settings.gradle` or `build.gradle`:

```groovy
dependencyResolutionManagement {
  repositories {
    google()
    mavenCentral()
    maven { url "https://jitpack.io" }
    maven { url 'https://developer.huawei.com/repo/' }
  }
}
```

Add the dependency to your app-level `build.gradle`:

```groovy
dependencies {
  implementation 'com.gameball:gameball-sdk:3.1.0'
}
```

<Info>
If your project uses older Gradle settings, replace `dependencyResolutionManagement` usage accordingly.
</Info>

### Maven

Add repositories to `pom.xml` then the dependency:

```xml
<dependency>
  <groupId>com.gameball</groupId>
  <artifactId>gameball-sdk</artifactId>
  <version>3.1.0</version>
</dependency>
```

## Initialize the SDK

Create an application-level initialization so the SDK is available globally.

<CodeGroup>
```kotlin Kotlin
class MyApplication : Application() {
    override fun onCreate() {
        super.onCreate()

        val config = GameballConfig.builder()
            .apiKey("your-api-key")      // Required
            .lang("en")                  // Required
            .platform("android")         // Optional (defaults to android)
            .shop("your-shop-id")        // Optional
            .sessionToken("your-session-token")    // Optional - for v4.1 endpoints
            .apiPrefix("custom-prefix")  // Optional - custom API prefix
            .build()

        GameballApp.getInstance(this).init(config)
    }
}
```

```java Java
public class MyApplication extends Application {
    @Override
    public void onCreate() {
        super.onCreate();

        GameballConfig config = GameballConfig.builder()
            .apiKey("your-api-key")      // Required
            .lang("en")                  // Required
            .platform("android")         // Optional (defaults to android)
            .shop("your-shop-id")        // Optional
            .sessionToken("your-session-token")    // Optional - for v4.1 endpoints
            .apiPrefix("custom-prefix")  // Optional - custom API prefix
            .build();

        GameballApp.getInstance(this).init(config);
    }
}
```
</CodeGroup>

## Configuration Parameters

<ParamField body="apiKey" type="string" required>
The client API key available in your Gameball dashboard (required).
</ParamField>

<ParamField body="lang" type="string" required>
The language code for SDK localization (e.g., "en", "ar", "fr"). Required.
</ParamField>

<ParamField body="platform" type="string">
Platform identifier, defaults to "android".
</ParamField>

<ParamField body="shop" type="string">
Your Gameball shop identifier for multi-shop configurations.
</ParamField>

<ParamField body="sessionToken" type="string">
Session Token for secure authentication (enables automatic v4.1 endpoint routing).
</ParamField>

<ParamField body="apiPrefix" type="string">
Custom API endpoint prefix.
</ParamField>

### Validation Rules

**GameballConfig requires:**
- API key is required (cannot be null)
- Language is required (cannot be null)

<Tip>
Make sure to register your Application class in AndroidManifest.xml:
```xml
<application
    android:name=".MyApplication"
    ...>
```
</Tip>

## Troubleshooting

### Common Issues

<AccordionGroup>
<Accordion title="SDK Not Initialized Error">
**Error:** `API key is required for customer initialization`

**Solution:** Ensure you call `GameballApp.getInstance(context).init(config)` before other SDK methods, preferably in your Application class's `onCreate()` method.
</Accordion>

<Accordion title="Invalid Customer ID Error">
**Error:** `Customer ID cannot be blank`

**Solution:** Provide a non-empty customer ID when building requests. The customer ID should be a permanent identifier for the user.
</Accordion>

<Accordion title="Network Errors">
**Symptoms:** Requests failing or timing out

**Solutions:**
- Check your internet connection
- Verify your API key is correct in the Gameball dashboard
- Ensure your server can reach Gameball endpoints
- Check if your app has `INTERNET` permission in AndroidManifest.xml
</Accordion>

<Accordion title="Push Notifications Not Working">
**Symptoms:** Device token not registered or notifications not received

**Solutions:**
- Verify both `deviceToken` and `pushProvider` are set during customer initialization
- Ensure Firebase/Huawei credentials are configured in Gameball dashboard
- Check that your Firebase/Huawei configuration is correct
- Test on a physical device (emulators may not receive push notifications)
</Accordion>
</AccordionGroup>

### Debug Logging

The SDK automatically enables debug logging in debug builds. To view SDK logs, filter by these tags:

| Log Tag | Purpose |
|---------|---------|
| `GameballApp` | General SDK operations and initialization |
| `LoggingInterceptor` | Network requests and responses |
| `HeaderInterceptor` | Request headers and authentication |

**Example using Logcat:**
```bash
adb logcat -s GameballApp LoggingInterceptor HeaderInterceptor
```

<Tip>
Network requests and responses are automatically logged when `BuildConfig.DEBUG` is true in your app. No additional configuration needed.
</Tip>
