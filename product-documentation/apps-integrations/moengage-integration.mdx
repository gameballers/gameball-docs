[MoEngage](https://www.moengage.com/) is an Intelligent Customer Engagement Platform that allows brands to personalize every customer interaction, driving better engagement, retention, loyalty, and lifetime value. The MoEngage and Gameball integration enables you to harness the full potential of both platforms to enhance your customer engagement strategies.

---

## Data Import Scenarios

The integration between Gameball and MoEngage ensures that your customer data is always up-to-date in MoEngage. Gameball's customer attributes will be imported into MoEngage whenever a customer is created or performs any action.

<AccordionGroup>
<Accordion title="Customer Attributes Sync to MoEngage" icon="user">
  **Trigger:** Upon customer creation, or after any new action by the customer (e.g., achievement, transaction, points expiry).

  **Description:** Gameball's customer attributes will be imported into MoEngage and updated once a customer is created or performs any action.

  **Imported Data:**

  | Attribute | Attribute |
  |---|---|
  | ID | Level Progress |
  | Email | Progress to achieve next level |
  | Join Date | Last order awarded points |
  | Mobile Number | Referral Code |
  | Birthday | Referral Link |
  | Gender | Tags |
  | Country | Segments |
  | Currency | RFM segments |
  | Points balance | Latest created coupon |
  | Pending points | Latest used coupon |
  | Current level name | Count of rewarded campaigns |
  | Current level order | Starting Store |
  | Next Level | Connecting Store |
  | Next level threshold | Store of First Transaction |
  | Average Basket | Number of Transactions |
  | Transactions Amount in the Last Rolling Year | Total Net Amount of Transactions |
  | Unit per Transaction | Earned Points |
  | Redeemed Points | Expired Points |
  | Date of First Transaction | Date of Last Transaction |
</Accordion>

<Accordion title="Update & Notify MoEngage with Every Order Placed" icon="bag-shopping">
  **Trigger:** "Order Placed" Event — Order placed by customer and received by Gameball.

  Once an order is placed by a customer, the following order attribute data is sent to MoEngage:

  - Order ID
  - Order Date
  - Total Paid
  - Total Price
  - TotalShipping
  - TotalTax
  - TotalDiscount
  - Discount Codes
  - Extra
  - Redeemed Amount
  - Rewarded Points
  - Equivalent Points Value
</Accordion>

<Accordion title="Update & Notify MoEngage with Every Coupon Created" icon="ticket">
  **Trigger:** "Coupon Created" Event — Coupon created for the customer.

  Once a coupon is created for the customer, the following coupon details will be available on MoEngage linked to the customer's data:

  - Coupon Code
  - Type
  - Value
  - Coupon Name
</Accordion>

<Accordion title="Update & Notify MoEngage with Every Coupon Used" icon="check-circle">
  **Trigger:** "Coupon Used" Event — Customer burnt the coupon while placing an order.

  Once a coupon is used by the customer, the following details will be available on MoEngage:

  - Coupon Code
  - Type
  - Value
  - Order ID
  - Coupon Name
  - Channel (POS, Online, etc.)
  - Order Quantity
</Accordion>

<Accordion title="Update & Notify MoEngage with Every Reward Campaign Achieved" icon="trophy">
  **Trigger:** "Reward Campaign Achieved" Event — Customer achieved a reward campaign.

  Once a Reward Campaign is achieved by the customer, the following details will be available on MoEngage:

  - Internal Name
  - Name
  - Reward (Points, Points & Score, Coupon Values)
  - Coupon Type (in case the reward was a coupon)
  - Equivalent Value of Rewarded points (in case the reward was in the form of points or Points & Score)
  - Achievement Date
</Accordion>

<Accordion title="Notify MoEngage upon Points Redemption" icon="coins">
  **Trigger:** "Points Redeemed" Event — Customer redeemed their saved points.

  Each `points_redeemed` event includes:

  - Redeemed points
  - Redemption amount
  - Redemption transaction ID (if available)
</Accordion>

<Accordion title="Update & Notify MoEngage with Customer Deletion" icon="trash">
  **Trigger:** Customer was removed from Gameball.

  Once a customer is deleted from Gameball, after **24 hours** the customer gets deleted from MoEngage.

  <Note>
  You have the option to disable the deletion feature from MoEngage.
  </Note>

  <Frame>
    <img src="/images/product-docs/apps-integrations/apps-integrations-26.png" />
  </Frame>
</Accordion>

<Accordion title="Notify MoEngage for Each Line Item Purchased" icon="list">
  **Trigger:** Order received by Gameball.

  Gameball sends an "Item Purchased" event call to MoEngage with all attributes of the item and the order ID:

  - orderId
  - Product Title
  - Product Price
  - Vendor Name
  - Product ID
  - Weight
  - Quantity
  - SKU
  - Collections array
  - Tags array
  - Categories array
  - "Extra" — any added attributes to this object will be sent inside the event
</Accordion>

<Accordion title="Workspace Information Synced to MoEngage" icon="building">
  Workspace context is included in both customer events and attributes, giving you greater flexibility in segmentation and personalization.

  - **`gameball_workspace`**: A string containing the workspace name or ID.
  - **Gameball Workspace**: A string attribute storing the associated workspace.
</Accordion>
</AccordionGroup>

## How to Set Up the Integration

<Steps>
<Step title="Navigate to the App Store on Gameball">
  From the Gameball dashboard, navigate to the **App Store**.
</Step>

<Step title="Select MoEngage">
  Select **MoEngage** from the list of available app integrations.

  <Frame>
    <img src="/images/product-docs/apps-integrations/apps-integrations-27.png" />
  </Frame>

  <Frame>
    <img src="/images/product-docs/apps-integrations/apps-integrations-28.png" />
  </Frame>
</Step>

<Step title="Enter Your MoEngage Credentials">
  Enter the **App ID and Data API Key** from MoEngage's settings page.

  <Frame>
    <img src="/images/product-docs/apps-integrations/apps-integrations-29.png" />
  </Frame>
</Step>

<Step title="Select Your Data Center">
  Select your **Data Center**.

  <Frame>
    <img src="/images/product-docs/apps-integrations/apps-integrations-30.png" />
  </Frame>

  <Note>
  If you're unsure which data center you're on, you can find your code on your dashboard's URL.

  <Frame>
    <img src="/images/product-docs/apps-integrations/apps-integrations-32.png" />
  </Frame>
</Step>

<Step title="Complete the Setup">

## How to Use the Data on MoEngage

Once the data from Gameball is imported into MoEngage, you can leverage it to create more personalized and effective customer engagement strategies:

<AccordionGroup>
<Accordion title="Personalized Campaigns" icon="envelope">
  Use customer attributes such as name, email, birthday, and purchase history to create highly personalized marketing campaigns. For instance, send birthday greetings with special discounts or recommend products based on past purchases.
</Accordion>

<Accordion title="Segmentation" icon="filter">
  Utilize the synced segments and tags to group customers based on their behavior, preferences, and activity. This allows you to target specific groups with tailored messages, increasing the relevance and impact of your communications.
</Accordion>

<Accordion title="Event-Triggered Campaigns" icon="bolt">
  Set up automated campaigns triggered by specific events such as order placement, coupon creation, or coupon usage. For example, send a thank-you email with a reward when a customer places an order or notify them of their new coupon.
</Accordion>

<Accordion title="Customer Journey Mapping" icon="map">
  Track and analyze the customer journey using the imported data to identify key touchpoints and optimize your engagement strategies. Understanding how customers interact with your brand at different stages helps in crafting more effective and timely interventions.
</Accordion>

<Accordion title="Retention Strategies" icon="heart">
  Monitor points balance, tier progress, and achievements to identify loyal customers and those at risk of churning. Create targeted campaigns to reward loyal customers and re-engage those who have shown signs of inactivity.
</Accordion>
</AccordionGroup>
