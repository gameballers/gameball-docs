## Why Integrate Segment with Gameball?

This integration will allow you to target and segment your customers based on how loyal they are to your app. Specifically, it enables you to:

1. Segment your contact lists based on Gameball customer attributes.
2. Trigger an automation based on Gameball customers' attributes.
3. Send more personalized emails by including Gameball customer information in email templates, for instance, the customer's current tier.

---

## How to Configure Segment on Gameball

<Steps>
<Step title="Add Events in the Gameball Admin Dashboard">
  Add the events you want to track in the **Gameball Admin Dashboard**.

  For more info on how to do that, [check out this article](/product-documentation/admin-settings/create-and-send-events). Here is an example:

  <Frame>
    <img src="/images/product-docs/apps-integrations/apps-integrations-89.png" />
  </Frame>
</Step>

<Step title="Add Gameball as a Destination in Segment">
  Add the Gameball app as a destination from the Segment catalog. Then add your Gameball API Key in the destinations settings on Segment. Learn [where to find your Gameball API key](/product-documentation/admin-settings/get-api-keys).

  <Frame>
    <img src="/images/product-docs/apps-integrations/apps-integrations-90.png" />
  </Frame>
</Step>

<Step title="Test with the Event Builder">
  Ensure that the events you track reach Gameball through the Event Tester on Segment with the Event Builder.

  <Frame>
    <img src="/images/product-docs/apps-integrations/apps-integrations-91.png" />
  </Frame>

  In the previous example, the Event name is (Buy product) and (type) shoes.

  <Note>
  The **Event Name** in Event Tester on Segment is the same Event Name in **Gameball Admin Dashboard**, and you must include `userId`.
  </Note>
</Step>

<Step title="Test with the JSON Builder">
  Ensure that the events you track reach Gameball through the Event Tester in Segment with the JSON Builder.

  <Frame>
    <img src="/images/product-docs/apps-integrations/apps-integrations-92.png" />
  </Frame>

  Event name is (Buy product) and (type) shoes.

  <Note>
  The only properties supported for updating through events at the moment are the ones in the customer Info box:

  - ID
  - Join Date
  - Level Name
  - Type ID
  - Points
  - Score Balance
  - Level ID
  - Completed Challenges
  </Note>
</Step>
</Steps>
