## Core & Platform

<AccordionGroup>
  <Accordion title="New 'Order Refunded' Event Available" icon="rotate-left">
    **Platform:** All | **Plan:** All | **Channel:** All

    A new system event — **`order_refunded`** — is now automatically triggered by Gameball whenever a refund is processed through the platform.

    <Note>
      This event is only applicable when using the **V4.0 Refund APIs**.
    </Note>

    **Use Case:** Track refunded orders directly within the customer profile, maintaining an accurate transaction history and improving customer support processes.

    **Key Benefits:**

    | Benefit | Description |
    |---------|-------------|
    | **Full Refund Visibility** | Every refund triggers an `order_refunded` event, logging essential refund data. |
    | **Customer Transparency** | Refunded orders are now visible within each customer's activity timeline. |
    | **Better Segmentation & Reporting** | Segment customers based on refund behavior or integrate this event into loyalty logic and analytics tools. |

    **Data Included in the Event:** Original order ID, Refund ID, Refunded Amount, Refunded Points.
  </Accordion>

  <Accordion title="API & Secret Key Rotation from Dashboard" icon="key">
    **Platform:** All | **Plan:** All | **Channel:** All

    Gameball has introduced the ability to **rotate your API keys and Secret Key individually** directly from the dashboard, offering enhanced security without integration disruptions. [Here is where to get your account integration details: API Key and Secret Key.](/product-documentation/admin-settings/get-api-keys)

    **Use Case:** Secure your accounts by rotating keys without affecting existing platform connections, helping comply with security best practices and mitigating potential key exposure risks.

    **Key Features:**

    - **Individual Key Rotation:** API keys and Secret Key can each be rotated separately.
    - **Immediate Replacement:** Upon rotation, a new key is generated instantly and the old key is immediately disabled.
    - **Automated Updates for Shopify & Salla:** Rotated keys are automatically updated in connected stores — no manual update needed.
    - **Access Control:** Only Admins, Workspace Owners, and Workspace Admins can rotate keys.
  </Accordion>

  <Accordion title="High Security Mode for API Authentication" icon="shield-halved">
    **Platform:** All | **Plan:** Shopify & Salla: Not available | Others: Available | **Channel:** All

    Gameball introduces **High Security Mode**, an optional feature that provides an extra layer of protection for your API integrations. When enabled, all API requests are verified and secure, preventing unauthorized access and data exposure. [Learn more about APIs and Secret Keys on Gameball](/product-documentation/admin-settings/get-api-keys)

    To activate this security option, go to **Settings > Admins Settings > Account Integration**, then scroll down to switch the toggle under **High Security Mode**.

    <Frame>
      <img src="/images/product-docs/product-releases/high-security-mode.png" alt="High security mode" />
    </Frame>

    **Benefits:**

    - **Enhanced Security:** Protects against unauthorized access by ensuring all API requests come from trusted sources.
    - **Data Privacy:** Prevents potential exposure of sensitive data.
    - **Improved Client Protection:** Encrypts IDs in client-side elements like widgets, making it impossible to reverse-engineer or misuse them.
  </Accordion>

  <Accordion title="Product-Level Order Segmentation" icon="filter">
    **Platform:** All | **Plan:** All | **Channel:** All

    Gameball has launched **Product-Level Order Segmentation**, enabling you to segment customers based on the exact products and quantities purchased in actual orders. [Learn more about segmentation](/product-documentation/customer-segmentation/how-to-segment-customers).

    **Use Case:** Create highly targeted and personalized campaigns by analyzing specific product purchase behavior — for example, target customers who purchased a specific SKU or segment customers buying a particular category or high-quantity product.

    **Key Features:**

    - **Product Line Item Filters:** Product ID, SKU, Category, Collection, Product Title, Quantity Purchased (per product), Price (per line item).
    - **Flexible Filtering Logic:** Filters can be combined using AND/OR conditions. Refunded and partially refunded orders are excluded by default for accuracy.
    - **System-Wide Availability:** Accessible across all dashboard areas where customer targeting is possible.
  </Accordion>

  <Accordion title="Custom User Permissions Per Invitation" icon="user-lock">
    **Platform:** All | **Plan:** All | **Channel:** All

    Gameball has released **Custom User Permissions**, giving Admins full control to configure dashboard access **per user individually** during the invitation process, without relying solely on preset roles. [Learn how to create and assign a new user role](/product-documentation/admin-settings/create-user-assign-roles).

    **Use Case:** Businesses can now grant team members (marketing, finance, CX) precisely the permissions they need, improving data security and operational control.

    <Frame>
      <img src="/images/product-docs/product-releases/custom-user-permissions.png" alt="Custom user permissions" />
    </Frame>

    **Key Features:**

    - **Granular Permissions at Invitation:** While inviting a new user, Admins can set permissions per section — Hide, Disabled, or Allow.
    - **Permissions Sections:** Customers, Campaigns, Coupons, Analytics, Settings, Notifications, Integrations, Billing, and more.
    - **Edit Permissions Anytime:** Admins can modify any user's permissions later from the User Management page.
    - **Security by Default:** New users are granted No Access by default and must be explicitly assigned permissions.
  </Accordion>
</AccordionGroup>
