## Core & Platform

<AccordionGroup>
  <Accordion title="Fixed Points Cashback Configuration Per Collection" icon="coins">
    **Platform:** All | **Plan:** Plans with Custom Collection-based Rules | **Channel:** Web, Mobile

    We've introduced a **new cashback reward type** that gives you more flexibility in how you reward customers: Configure cashback rules to issue **fixed points per collection.** [Learn more about giving different cashback values for different collections](/product-documentation/programs/loyalty-points-earn/collection-based-earning-rules).

    **Use Case:** Ideal if you want to offer consistent point rewards per collection, regardless of the product's price. For example, reward 100 points per product in the "New Arrivals" collection whether an item costs $20 or $200.

    **Configuration:** While setting up or editing Custom Cashback Rules, choose **Fixed Points (new)**.

    <Frame>
      <img src="/images/product-docs/product-releases/fixed-points-cashback-collection.png" alt="Fixed points cashback per collection" />
    </Frame>

    **API Requirements** — send the following details **per item** in the Order API:

    | Field | Required |
    |-------|----------|
    | Collection ID(s) | Yes |
    | Price | Yes |
    | Discount | Yes |
    | Taxes | Yes |
    | Product ID | Yes |

    **Key Rules:**

    - If an item belongs to multiple collections → Gameball rewards based on the **highest fixed point value**.
    - If multiple different items from the same collection are purchased → Fixed points reward applies per item.
    - The **fixed points reward still respects active Points Multiplier Campaigns** (e.g., Double Points → 200 becomes 400).
  </Accordion>

  <Accordion title="Customizable Referral Page Image" icon="image">
    **Platform:** All | **Plan:** All | **Channel:** Web, Mobile

    We have added a new feature that allows you to **customize the referral page image directly from the dashboard**, giving you greater control to match the referral experience with your unique brand identity. [Learn how to control your widget visibility and images](/product-documentation/customer-widget/start-configuring-profile/customizing-tabs-visibility-and-order-in-gameball-widget-settings).

    **Image Options:**

    | Option | Description |
    |--------|-------------|
    | Default Image | Use Gameball's pre-set image |
    | Custom Image | Upload your own branded image |
    | No Image | Display no image at all on the referral page |

    **Dashboard Location:** Navigate to `Settings > General > Visibility > Referral Page`

    **Image Guidelines:**

    - Supported formats: `.jpg`, `.png`
    - Recommended size: **280 x 280 px**
    - Max file size: **1MB**

    <Frame>
      <img src="/images/product-docs/product-releases/customizable-referral-page-image.png" alt="Customizable referral page image" />
    </Frame>
  </Accordion>

    **Platform:** All | **Plan:** All | **Channel:** Web, Mobile

    We have enhanced the widget branding experience by allowing you to upload **custom Earn and Redeem icons**, giving you greater control over the look and feel of your loyalty widget. ## Other Improvements

    Gameball has improved the **Dynamic Reward Experience** in the Reward Campaign setup. Now, when you try to select a **Dynamic Reward** and it's disabled, a **tooltip** will automatically appear to explain why the option is not available.

    **The tooltip explains that Dynamic Rewards are disabled when:**

    - No event is selected.
    - Multiple events are selected.
    - The selected event is set to trigger multiple times.
