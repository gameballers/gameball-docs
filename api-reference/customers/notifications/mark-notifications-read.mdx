## Mark Notifications as Read

Mark one or more notifications as read for the specified customer.

<Info>
Security: Provide both `apikey` and `secretkey` headers.
</Info>

<Note>
Concepts and examples adapted from the official Gameball Customers documentation: https://developer.gameball.co/rest-api/customers
</Note>

---

### Overview

Use this endpoint to update notification read state and power badge counters.


### Parameters

Path

<ParamField path="customerId" type="string" required>
Unique identifier of the customer.
</ParamField>

Body

<ParamField body="notificationIds" type="array[string]" required>
List of notification IDs to mark as read.
</ParamField>

---

### Request

<RequestExample>
```bash cURL
curl -X POST "https://api.gameball.co/api/v4.0/integrations/customers/1848877205/notifications/read" \
  -H "Content-Type: application/json" \
  -H "apikey: YOUR_API_KEY" \
  -H "secretkey: YOUR_SECRET_KEY" \
  -d '{"notificationIds":["n_01HZYF8Q7N8R","n_01HZYF8Q7P2C"]}'
```
</RequestExample>

### Response

<ResponseExample>
```json Success
{ "success": true, "updated": 2 }
```
</ResponseExample>
